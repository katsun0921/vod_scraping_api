# 劇場情報源 候補リスト（日本限定・初期調査）

作成: 2026-07-17
対象: `docs/feature/theater-release-calendar-spec.md` レイヤー1データソース（未確定事項#1）

## 調査方法・限界

このセッションのネットワークポリシーにより外部サイトへの直接アクセスが全面ブロックされている
（`WebFetch`・`curl`ともに主要ドメイン宛の接続が403で拒否される。`example.com`宛でも同様のため
サイト固有のブロックではなく環境側の制限）。そのため以下は**Web検索のスニペットのみ**に基づく
一次調査であり、以下は未検証。

- RSSフィードが実際に稼働しているか
- フィードの中身が「公開日付き作品一覧」として構造化されているか、単なるサイト更新通知か
- 利用規約の正確な文言・スクレイピング可否

**「劇場情報源」シートに登録する前に、必ず人間がブラウザ・RSSリーダーで実際に開いて内容と
利用規約を確認すること**（仕様書7.「利用条件：利用規約を確認する」に対応）。

## A. 推奨（構造化API・ライセンスが明確）

| 候補 | 概要 | 取得方式 | 備考 |
|---|---|---|---|
| **TMDb API** `/discover/movie`（`region=JP` + `with_release_type=2,3`）or `/movie/{id}/release_dates` | 公式APIで劇場公開日（type=2先行公開/3劇場公開）を国別に取得できる | api（要`TMDB_API_KEY`） | 利用規約が明確で、既存 `docs/feature/coming-soon-pipeline.md` で配信系データ用に採用予定のTMDb基盤を流用できる。ミニシアター・単館系邦画の網羅性は未検証で、大型配給作品中心になる可能性が高い |

TMDbは仕様書の非ゴール「全配給会社・全ミニシアター作品の完全網羅を保証しない」と初期MVPの
方針に合致しやすく、法的リスクが最も低い。**着手優先度としてはこれを最有力候補として推奨**。

### TMDb APIの費用（要最終確認）

| 利用形態 | 費用 |
|---|---|
| 非商用利用 | 無料。TMDB出典表示（attribution）義務あり |
| 商用利用（広告収益等でマネタイズしている場合） | 別途ライセンス契約が必要。年間売上$1M未満は $149/月（Commercial APIプラン）、$1M超は個別見積もり |
| レート制限 | 従量課金ではなくIPベースで秒間約40リクエストの上限のみ |

**注意**: 「広告表示によるサイトへの集客・収益化」を商用利用とみなす、という公式記述がある一方、
TMDB担当者のフォーラム回答では「広告自体は問題ない」という揺れた発言も見られ、線引きが曖昧。
Katsumascoreが何らかの形でマネタイズしている場合は商用利用に該当する可能性があるため、
`api@themoviedb.org` へ問い合わせて確認することを推奨する。この判定は
`docs/feature/coming-soon-pipeline.md`（同じ`TMDB_API_KEY`を使う想定）にも影響する。

リスクを踏まえると、A・Bの検証が完了するまでは後回しにすべき候補。

## 次のアクション（人間側での確認事項）

1. TMDb APIキーを取得済みか確認する（`docs/feature/coming-soon-pipeline.md` 未決定事項#1と共通で、
   取得済みなら劇場公開日取得にも流用できる）。Katsumascoreが商用利用に該当するか
   （広告収益等でマネタイズしているか）を確認し、該当する場合は`api@themoviedb.org`へ
   ライセンス要否を問い合わせる（A.のTMDb APIの費用セクション参照）
2. A・Bの候補について、実際にブラウザ・RSSリーダーで内容を開いて確認する
   - 映画.comは利用規約の複製禁止条項があるため、運営への確認が望ましい
3. 確認が取れたソースを「劇場情報源」シート（`ID / 名称 / URL / 取得方式 / レイヤー / 有効/無効 /
   規約確認済み / メモ`）に追加する。「規約確認済み」="済" にした行のみ `fetch_theater.py` の
   取得対象になる
4. `取得方式=api`（TMDb等）は `fetch_theater.py` 未対応のため、A案を採用する場合は実装が必要
   （現状`rss`のみ対応。仕様書17.のTODO#10と同じ課題）
